# VKR_Russkin

Веб-приложение для построения дымовой трассы и расчета потерь давления в ее элементах. Пользователь формирует маршрут из прямых участков, поворотов, сужений, расширений и местных сопротивлений, задает параметры газа и геометрию, после чего получает подробный расчет, итоговую таблицу, графики и рекомендации.

## Возможности

- графическое построение маршрута дымовых газов;
- круглые и прямоугольные поперечные сечения;
- расчет скорости, числа Рейнольдса и коэффициента трения;
- расчет потерь давления на трение и местных сопротивлениях;
- учет геометрического давления по общему перепаду высот трассы;
- автоматический выбор КМС по справочным таблицам и диаграммам;
- линейная, билинейная и последовательная многомерная интерполяция;
- предупреждения при выходе параметров за область применения зависимостей;
- сохранение расчетов и пользовательских заготовок;
- формирование отчета, графиков и версии для печати;
- запуск с SQLite или MariaDB;
- контейнерное развертывание через Docker Compose;
- автоматизированная сборка и развертывание через Jenkins.

## Запуск через Docker Compose

1. Создать локальный файл окружения:

```powershell
Copy-Item .env.example .env
```

2. Заменить демонстрационные пароли в `.env` на случайные значения:

```dotenv
MARIADB_PASSWORD=change-this-password
MARIADB_ROOT_PASSWORD=change-this-root-password
```

3. Проверить конфигурацию и запустить сервисы:

```powershell
docker compose config
docker compose up -d --build
```

4. Открыть приложение:

```text
http://localhost:5454/
```

5. Проверить контейнеры и журнал приложения:

```powershell
docker compose ps
docker compose logs --tail 100 app
```

Остановка контейнеров:

```powershell
docker compose down
```

Удаление контейнеров вместе с данными MariaDB:

```powershell
docker compose down -v
```

Команда с параметром `-v` удаляет Docker volume базы данных и должна применяться только тогда, когда сохраненные данные больше не нужны.

## Использование

1. Задать начальную температуру, расход и состав дымовых газов.
2. Выбрать материал газохода и состояние поверхности либо ввести собственную эквивалентную шероховатость.
3. Добавить на холст элементы трассы в фактической последовательности движения газа.
4. Для каждого элемента указать форму и размеры сечения, длину, теплопотери и специальные параметры.
5. При необходимости включить учет геометрического давления и задать общий перепад высот.
6. Проверить предупреждения о геометрии и диапазонах справочных зависимостей.
7. Выполнить расчет и открыть итоговый отчет.
8. При авторизации сохранить расчет или создать заготовку маршрута.

Положительный перепад высоты означает, что конечная точка трассы расположена выше начальной, отрицательный - ниже начальной.

## Расчетная модель

Для каждого элемента последовательно определяются:

- средняя температура газа;
- плотность и кинематическая вязкость;
- площадь и эквивалентный диаметр сечения;
- скорость газа;
- число Рейнольдса;
- коэффициент трения;
- коэффициент местного сопротивления;
- потери давления на трение и местном сопротивлении.

Потери на трение рассчитываются по зависимости Дарси-Вейсбаха:

```text
ΔPтр = λ · L / Dэкв · ρw² / 2
```

Потери на местном сопротивлении:

```text
ΔPмс = ζ · ρw² / 2
```

Геометрическое давление рассчитывается один раз для всей трассы и не относится к отдельному блоку:

```text
ΔPгеом = g · H · (ρгаза - ρвоздуха)
```

Итоговое сопротивление трассы:

```text
ΣΔP = ΣΔPтр + ΣΔPмс + ΔPгеом
```

## База данных

Локально используется SQLite. В Docker Compose приложение подключается к MariaDB, а справочные данные при первом запуске копируются из поставляемой SQLite-базы.

Основные группы таблиц:

| Назначение | Таблицы |
|---|---|
| КМС и двумерные таблицы | `LRCs`, `ResistanceDataPoints` |
| Вязкость и плотность компонентов | `KVoAs`, `DotMCs` |
| Эквивалентная шероховатость | `RoughnessMaterials`, `RoughnessConditions`, `Roughnesses` |
| Пользователи и сессии | `AppUsers`, `UserSessions` |
| Заготовки | `UserPresets` |
| Сохраненные расчеты | `SavedCalculations` и связанные таблицы |

В поставляемой обезличенной базе находятся:

- 22 записи каталога местных сопротивлений;
- 9 табличных местных сопротивлений;
- 388 расчетных точек КМС;
- 13 значений кинематической вязкости;
- 5 значений плотности компонентов;
- 99 сочетаний материала и состояния поверхности;
- 0 пользователей, сессий и сохраненных расчетов.

## Где хранятся справочные зависимости

Справочные данные разделены на три вида.

### Постоянные КМС

Постоянные значения хранятся в `LRCs.ValueofLR`. Например, так представлены задвижка, составное колено и некоторые типовые сопротивления.

### Двумерные таблицы в базе данных

Таблица `ResistanceDataPoints` хранит точки вида:

```text
ResistanceId, ParamX, ParamY, ZetaValue
```

Так представлены:

- конический коллектор;
- диаграммы 3-1, 3-2 и 3-3;
- варианты диаграммы 3-4;
- диаграмма 3-5;
- диаграммы 3-6 и 3-7.

Для каждой зависимости база содержит полную прямоугольную сетку. Между соседними точками выполняется билинейная интерполяция. Одномерные таблицы представлены сеткой, в которой одна из осей содержит единственное значение.

### Многомерные диаграммы в C#

Не все диаграммы находятся в базе данных:

| Диаграмма | Реализация |
|---|---|
| 3-1...3-7 | таблицы `LRCs` и `ResistanceDataPoints` |
| 4-1 | расчетная формула внезапного расширения |
| 5-2 | многомерные массивы в `PressureCalculationService` |
| 5-4 | многомерные массивы в `PressureCalculationService` |
| 5-23 | двумерная матрица в `PressureCalculationService` |
| 5-27 | две одномерные таблицы и расчетные формулы в `PressureCalculationService` |
| 5-28 | геометрические формулы и таблицы диаграммы 5-4 |
| 6-1 | несколько таблиц и расчетные формулы в `PressureCalculationService` |

Массивы 5-2, 5-4, 5-23, 5-27 и 6-1 нельзя удалить из кода без предварительного переноса данных: соответствующих числовых наборов в текущей базе нет.

Текущая структура `ResistanceDataPoints` рассчитана только на две координаты `ParamX` и `ParamY`. Она не может без потери смысла представить:

- три и более независимые оси;
- несколько режимов одной диаграммы;
- набор отдельных поправочных коэффициентов;
- зависимость результата от формул, шероховатости и числа Рейнольдса.

Хранение этих массивов в C# допустимо, поскольку данные являются статическими, изменяются вместе с расчетным алгоритмом и проверяются при запуске приложения. Браузер получает те же массивы от сервера через конфигурацию страницы, поэтому второй независимой копии таблиц в JavaScript нет.

## Возможный перенос диаграмм из кода

Для данного проекта наиболее простой вариант - хранить многомерные диаграммы в версионируемых JSON-файлах, например:

```text
VKR_Russkin/ReferenceData/
|-- diagram-5-2.json
|-- diagram-5-4.json
|-- diagram-5-23.json
|-- diagram-5-27.json
`-- diagram-6-1.json
```

При запуске файлы должны загружаться в строго типизированные модели и проходить те же проверки осей, размеров матриц и числовых значений. Формулы интерполяции и физические поправки при этом останутся в `PressureCalculationService`.

Если требуется редактирование диаграмм через базу данных, понадобится расширенная схема:

- `EngineeringDiagrams` - описание диаграммы, версия и источник;
- `DiagramAxes` - названия, единицы и порядок осей;
- `DiagramAxisValues` - узлы каждой оси;
- `DiagramTables` - отдельные режимы или подтаблицы;
- `DiagramCells` - значения в узлах многомерной сетки;
- `DiagramApplicabilityRules` - ограничения и область применения.

Даже после такого переноса расчетные формулы 5-27, 5-28 и 6-1 должны остаться в коде, поскольку база хранит данные, но не должна исполнять физическую логику.

## Интерполяция и выход за диапазон

- Для одномерных зависимостей применяется линейная интерполяция.
- Для двумерных таблиц применяется билинейная интерполяция.
- Для многомерных диаграмм последовательно интерполируются соседние таблицы.
- Размеры осей и матриц проверяются при запуске.
- Полнота двумерных сеток базы также проверяется при запуске.
- Значение ограничивается ближайшей границей только в случаях, предусмотренных логикой конкретной диаграммы.
- В остальных случаях расчет блокируется с сообщением о допустимом диапазоне.
- При использовании граничного значения пользователь получает предупреждение.

## Конфигурация

Локальная конфигурация SQLite находится в `VKR_Russkin/appsettings.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Data Source=VKR_Russkin.db"
  }
}
```

Для MariaDB используются переменные окружения Docker Compose. Пароли не должны записываться в `docker-compose.yml`, `Jenkinsfile` или отслеживаемые файлы конфигурации.

## CI/CD

`Jenkinsfile` выполняет:

1. получение исходного кода;
2. восстановление NuGet-зависимостей;
3. сборку Release;
4. запуск тестовых проектов при их наличии;
5. проверку Docker Compose;
6. сборку контейнерных образов;
7. развертывание сервисов;
8. проверку состояния контейнеров и HTTP endpoint `/health`.

На сервере Jenkins использует внешний файл:

```text
/etc/vkr-russkin/deploy.env
```

Этот файл не входит в репозиторий и должен быть доступен только учетной записи, выполняющей развертывание.
